Customer Churn Prediction

Problem Telecom companies lose revenue when customers leave without warning. Predicting churn before it happens allows retention teams to act proactively instead of reactively.
Solution Built an end-to-end supervised learning pipeline on the IBM Telco dataset (7,043 records). Covered raw ingestion → EDA → feature engineering on usage patterns → class imbalance handling → model training and evaluation using accuracy, F1-score, and AUC-ROC curve analysis.
Impact Model outputs both binary churn labels and probability scores, making predictions actionable for business teams.

Search Algorithm Optimizer using ML

Problem No single graph search algorithm is universally optimal. BFS, DFS, A*, and Hill Climbing each perform differently depending on graph structure — but choosing the right one manually is guesswork.
Solution Built a machine learning system that analyses graph properties (density, size, branching factor) and performance metrics from benchmarks to predict the optimal algorithm for a given graph. Used NetworkX to construct diverse graph structures and Scikit-learn for the recommendation model.
Impact Produces structured comparison tables and per-algorithm profiling — turning algorithm selection from intuition into a data-driven decision.
↗ BFS · DFS · A* · Hill Climbing ↗ ML-Powered Recommendation ↗ Graph Property Analysis
